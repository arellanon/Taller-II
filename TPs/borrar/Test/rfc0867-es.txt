                        Protocolo de hora diaria

Este RFC especifica un estándar para comunidad ARPA en Internet. Es de
esperar que los hosts en ARPA Internet que elijan implementar un
protocolo de hora diaria adopten e implementen este estándar.

El servicio de hora diaria es una útil herramienta de depuración y
medida. Un servicio de hora diaria envía la fecha y hora actual como una
cadena de caracteres sin tener en cuenta la entrada.

Servicio de hora diaria basado en TCP

   Un servicio hora diaria se define como una aplicación basada en una
   conexión TCP. Un servidor espera conexiones TCP en el puerto TCP 13.
   Una vez se ha establecido una conexión se envía la fecha y hora
   actual a través de la conexión como una cadena de caracteres ascii (y
   cualquier dato recibido es descartado). El servicio cierra la
   conexión una vez enviada la cadena.

Servicio de hora diaria basado en UDP

   Otro servicio de hora diaria se define como una aplicación basada en
   datagramas sobre UDP. Un servidor espera datagramas UDP en el puerto
   UDP 13. Cuando se recibe un datagrama, se envía un datagrama de
   respuesta conteniendo un la fecha y hora actual como una cadena de
   caracteres ascii (los datos del datagrama recibido se ignoran).

Sintaxis de la hora diaria

   No hay una sintaxis específica para la hora diaria. Se recomienda
   limitarse a caracteres ASCII imprimibles, espacios, retornos de carro
   y avances de línea. La hora diaria debe ocupar una sola línea.

Una sintaxis muy usada es:

     Día de la semana, Mes Día, Año Zona-horaria

     Ejemplo

       Jueves, 22 de febrero, 1982 17:37:43-PST

   Otra sintaxis muy usada en la que se utiliza en SMTP:

     dd mmm yy hh:mm:ss zzz
